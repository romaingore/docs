## Modèle OSI (simplifié)

| Couche | Nom | Protocoles | Rôle |
|---|---|---|---|
| 7 | Application | HTTP, DNS, SSH | Interface utilisateur |
| 4 | Transport | TCP, UDP | Fiabilité, ports |
| 3 | Réseau | IP, ICMP | Adressage, routage |
| 2 | Liaison | Ethernet, Wi-Fi | Transmission locale |
| 1 | Physique | Câble, fibre | Signal brut |

## Adressage IP

```
IPv4 : 192.168.1.100/24
        │           │
        Adresse     Masque (24 bits = 255.255.255.0)

Réseau  : 192.168.1.0
Broadcast: 192.168.1.255
Hôtes    : 192.168.1.1 à 192.168.1.254 (254 hôtes)
```

## DNS

Le **DNS** (Domain Name System) traduit les noms de domaine en adresses IP.

```
navigateur → cache local → resolver → root → TLD (.com) → autoritaire
```

### Types d'enregistrements

| Type | Rôle | Exemple |
|---|---|---|
| **A** | Nom → IPv4 | `example.com → 93.184.216.34` |
| **AAAA** | Nom → IPv6 | `example.com → 2606:2800:...` |
| **CNAME** | Alias | `www.example.com → example.com` |
| **MX** | Serveur mail | `example.com → mail.example.com` |
| **NS** | Serveur DNS | `example.com → ns1.example.com` |

## Commandes réseau

```bash
ip addr                    # interfaces réseau
ping -c 4 google.com       # tester la connectivité
traceroute google.com      # tracer la route
dig example.com            # requête DNS
nslookup example.com       # résolution DNS
curl -v https://api.com    # requête HTTP (verbose)
ss -tulnp                  # ports ouverts
netstat -an                # connexions réseau
```
